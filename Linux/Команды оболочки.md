
- **kill** - убить процесс
```bash
kill 1234                 # SIGTERM (корректное завершение)
kill -9 1234              # SIGKILL (принудительное завершение)
kill -s HUP 5678          # Перезагрузка конфигурации (часто используется в Nginx, SSHd)
kill -l                   # Показать список всех сигналов
kill -l 9                 # Показать имя сигнала №9 (KILL)
kill -p 1234              # Проверить PID, не трогая процесс
kill -s CONT 1234         # Возобновить приостановленный процесс (SIGCONT)
kill -15 $(pgrep python3) # Отправить SIGTERM всем процессам python3
```

- **ss** - Современная замена `netstat`. Показывает процессы и их PID.
```bash
sudo ss -tulnp | awk '$4 ~ /:8080$/'
```
- `-t` / `-u` → TCP / UDP
- `-l` → только слушающие порты (уберите, если ищете установленные соединения)
- `-n` → числовой вывод (без резолвинга имён)
- `-p` → показать процессы (`users:(("nginx",pid=1234,fd=6))`)

 - **lsof** - показывает открытые файлы и сетевые сокеты
 ```bash
 sudo lsof -i :8080
 ```

```bash
sudo lsof -ti :8080   # Получить только PID
# Вывод: 1234
```

- **fuser** - Специализированная утилита для поиска процессов, использующих файлы/порты.
```bash
sudo fuser 8080/tcp
# Вывод: 1234 5678
```
- `/tcp` или `/udp` → протокол (обязательно указывать)
- `-v` → подробный вывод с пользователем и именем команды
- `-k` → **убить** процесс(ы), занимающие порт
