
**Директивы компилятора** — это специальные комментарии в исходном коде, которые дают инструкции компилятору или инструментам Go для изменения поведения сборки, оптимизации или генерации кода

#### Требования к формату написания
Для всех директив `//go:` действуют строгие правила:
```go
//go:directive_name [аргументы]
```


| Требование                 | Описание                                                                                                                 |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| ✅ **Начало строки**       | Директива должна начинаться с `//go:` без пробела между `//` и `go:`                                                     |
| ✅ **Отдельная строка**    | Каждая директива должна быть на своей строке                                                                             |
| ✅ **Только пробелы/табы** | Перед комментарием допускаются только пробелы или табуляция                                                              |
| ✅ **Позиция**             | Директива должна располагаться **непосредственно перед** элементом, к которому применяется (для функциональных директив) |
| ✅ **Импорт unsafe**       | Некоторые директивы (например, `//go:linkname`) требуют `import _ "unsafe"` в файле                                      |

> ⚠️ **Важно**: Если между директивой и функцией будет пустая строка — директива **не сработает**


#### Основные категории директив

##### 1. Директива сборки (//go:build)

**Назначение**: Условная компиляция файлов в зависимости от ОС, архитектуры или кастомных тегов.
**должна быть первой** — всё, что выше, должно быть только комментариями или пустыми строками
```go
//go:build linux && amd64
// +build linux,amd64  // устаревший синтаксис
```

```go
//go:build linux
package main

func init() {
    // Этот код скомпилируется только для Linux
    println("Running on Linux")
}
```



##### 2. Функциональные директивы
Применяются к функции, которая следует **непосредственно после** директивы

| Директива             | Описание                                                                                                                             |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `//go:noinline`       | Запрещает компилятору встраивать функцию.                                                                                            |
| `//go:nosplit`        | Отключает проверку переполнения стека. **Опасно**: Используйте только в runtime-коде. Может привести к повреждению памяти.           |
| `//go:noescape`       | Указывает, что указатели, переданные в функцию, не "убегают" в кучу. Использование: компилятор может оптимизировать выделение памяти |
| `//go:uintptrescapes` | Указывает, что аргументы `uintptr` могут быть указателями, которые должны оставаться в куче.                                         |
| `//go:norace`         | Исключает функцию из проверки race detector.                                                                                         |
|                       |                                                                                                                                      |


##### 3. Директива `//go:linkname`
Позволяет связать локальный символ с символом из другого пакета (включая неэкспортированные).
Требует `import _ "unsafe"` - иначе ошибка компиляции.
```go
**package mypkg

import _ "unsafe"  // обязательно!

//go:linkname timeNow runtime.nanotime
func timeNow() int64  // только объявление**
```

> ⚠️ **Предупреждение**: `//go:linkname` нарушает инкапсуляцию и может сломаться при обновлении стандартной библиотеки. Используйте с крайней осторожностью.



##### 4. Директива `//go:generate`
Запускает внешние команды для генерации кода перед сборкой
```go
//go:generate go run gen.go -output types.go
//go:generate stringer -type=Status
//go:generate mockgen -source=interface.go -destination=mocks/interface_mock.go

package main
```

**Запуск**:
```bash
go generate ./...  # выполнит все директивы в пакете
```

**Популярные инструменты**:

- `stringer` — генерация `String()` методов для enum
- `mockgen` — создание моков для интерфейсов
- `protoc` — генерация кода из protobuf-файлов

##### 5. WebAssembly-директивы (GOOS=wasip1)

- `//go:wasmimport` - Импортирует функцию из WebAssembly-модуля
```go
//go:wasmimport env log
func wasmLog(ptr uint32, size uint32)  // только объявление
```

- `//go:wasmexport` - Экспортирует Go-функцию для вызова из хоста:
```go
//go:wasmexport add
func Add(a, b int32) int32 {
    return a + b
}
```


##### 6. Новая директива в Go 1.26: `//go:fix`
Директива для автоматического рефакторинга кода через `go fix`
**Применение**: миграция кода при обновлении библиотек, автоматизация рефакторинга.
```go
//go:fix inline oldpkg.OldFunc newpkg.NewFunc

// После выполнения `go fix`:
// oldpkg.OldFunc() → newpkg.NewFunc()
```
	

#### Места применения директив в исходном коде Go

```
📄 файл .go
│
├─ [0] Пустые строки / обычные комментарии (опционально)
│
├─ [1] //go:build  ← ТОЛЬКО ЗДЕСЬ
│   └─ // +build (устаревший синтаксис)
│
├─ [2] //go:generate (опционально, можно несколько)
│
├─ [3] package main  ← ОБЯЗАТЕЛЬНО после [1] и [2]
│
├─ [4] Импорт "unsafe" (если нужны linkname/noescape и др.)
│
├─ [5] Объявления переменных / функций
│   │
│   ├─ //go:embed  ← перед var или внутри func
│   │
│   ├─ //go:noinline  ← НЕПОСРЕДСТВЕННО перед func
│   │   func Foo() { ... }
│   │
│   ├─ //go:linkname  ← перед func/var (позиция не критична)
│   │   func bar()  // без тела
│   │
│   └─ //line  ← в любом месте (для генераторов кода)
│
└─ [6] Остальной код
```




|Тип директивы|Где размещается|Требования к позиции|
|---|---|---|
|`//go:build`|**В начале файла**, до `package`|Только пустые строки/комментарии выше|
|`//go:generate`|**Перед `package`**|В любом месте до объявления пакета|
|`//go:binary-only-package`|**В начале файла**, до `package`|Как `//go:build`|
|`//go:noinline`, `//go:nosplit`, `//go:noescape`, `//go:norace`, `//go:uintptrescapes`|**Непосредственно перед функцией**|Без пустых строк между директивой и `func`|
|`//go:linkname`|Перед `var`/`func` (рекомендуется)|Позиция не критична для работы, но должна быть в том же файле|
|`//go:embed`|**Внутри функции** или перед `var`|В любом месте, где допустимы объявления|
|`//go:build tags` (в имени файла)|**В имени файла**|Например: `service_linux.go`|
|`//line`|**В любом месте**|Историческая директива для генераторов кода|
|`//go:wasmimport` / `//go:wasmexport`|Перед функцией (WASM-сборка)|Только для GOOS=wasip1|
