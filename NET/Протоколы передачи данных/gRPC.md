## Тулинг для работы с gRPC

#### protoc-gen-go

**Что он генерирует:**
- **Go-структуры:** Для каждого `message` создается соответствующий `struct`.
- **Методы сериализации:** Методы для превращения структуры в байты (protobuf format) и обратно (`Marshal`, `Unmarshal`).
- **Вспомогательные методы:** Геттеры, методы `String()`, `Reset()`, `ProtoReflect()` и т.д.
- **Файл:** Обычно создает файл с суффиксом `.pb.go` (например, `user.pb.go`).

**Зачем нужен:** Даже если вы **не используете gRPC**, этот плагин нужен, если вы хотите использовать Protobuf для:

- Сериализации данных (например, для сохранения в кэш Redis).
- Обмена сообщениями через очереди сообщений (Kafka, RabbitMQ).
- Валидации данных.

####  protoc-gen-go-grpc
Этот плагин преобразует определения `service` из `.proto` файла в Go-код для удаленных вызовов.

**Что он генерирует:**
- **Интерфейс сервера:** Go-интерфейс, который вы должны реализовать на стороне сервера, чтобы обрабатывать запросы.
- **Клиентская заглушка (Stub):** Структура клиента, через которую вы вызываете методы удаленного сервера как обычные Go-функции.
- **Функции регистрации:** Функция (обычно `Register...Server`), которая связывает ваш реализованный сервер с gRPC-сервером.
- **Файл:** Обычно создает файл с суффиксом `_grpc.pb.go` (например, `user_grpc.pb.go`).

**Зачем нужен:** Только если вы используете **gRPC** для сетевого взаимодействия между сервисами.

## УСТАНОВКА КОМПОНЕНТОВ
#### Protoc
- [https://github.com/protocolbuffers/protobuf/releases](https://github.com/protocolbuffers/protobuf/releases)- скачать нужный релиз софтины. Для этого:
- выбираем последнюю версию на старице
- прокручиваем вниз до пояления списка `assets`
- выбираем пункт для нужной ОС и качаем архив
- архив подкладываем в нужную папку
-  В системной переменной `PATH` установить абсолютный путь к директории /bin протобафа
- проверить установку протобафа: 
```bash
 protoc --version
```

#### Плагины для protoc:
```go
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest
```
Убедиться что путь, эквивалентный `$GOPATH/bin` находится в переменной `$PATH`.

## Компиляция proto-файла

```bash

protoc -I proto proto/sso/sso.proto --go_out=./gen/go/ --go_opt=paths=source_relative --go-grpc_out=./gen/go/ --go-grpc_opt=paths=source_relative
```

Пояснения к командам:
- `-I proto`: Опция `-I` или `--proto_path` указывает путь к корневой директории с .proto файлами. Это нужно для того, чтобы компилятор смог найти импорты, если они есть. В нашем случае это директория `proto`.
- `proto/sso/sso.proto`: путь к конкретному .proto файлу, который мы компилируем.
- `--go_out=./gen/go/`: запускает `protoc-gen-go`. Опция `--go_out` указывает, куда записывать сгенерированный Go-код. В нашем случае — `./gen/go/`.
- `--go_opt=paths=source_relative`: дополнительная опция — указывает, как создавать имена пакетов. `paths=source_relative` означает, что выходные файлы будут иметь тот же пакет, что и исходные .proto файлы.
- `--go-grpc_out=./gen/go/`:запускает `protoc-gen-go-grpc`.Указывает куда записывать сгенерированный Go gRPC-код. Как и в предыдущем случае, выходные файлы будут помещены в директорию `./gen/go/`.
- `--go-grpc_opt=paths=source_relative`: Это аналогичная опция для генерации Go gRPC-кода, указывающая, как создавать имена пакетов для gRPC.
