### 1. **Паттерны интеграции (Integration Patterns)**

Решают задачи взаимодействия между системами, особенно в распределённых архитектурах.
- Примеры:
    - **Message Broker**, **Request-Reply**, **Publish-Subscribe**, **Dead Letter Channel**, **Claim Check**
- Источник: книга _Enterprise Integration Patterns_ (Hohpe & Woolf)
> Уровень: между сервисами, очередями, шинами сообщений.


### 2. **Паттерны развертывания / DevOps-паттерны (Deployment / Operational Patterns)**

Описывают стратегии доставки, масштабирования, мониторинга и управления жизненным циклом приложений.
- Примеры:
    - **Blue-Green Deployment**, **Canary Release**, **Feature Toggle**, **Sidecar**, **Service Mesh**, **Health Check API**
> Уровень: инфраструктура, CI/CD, эксплуатация.

### 3. **Паттерны данных (Data Patterns)**

Фокусируются на хранении, обработке и управлении данными.
- Примеры:
    - **Event Sourcing**, **CQRS (Command Query Responsibility Segregation)**, **Database per Service**, **Change Data Capture (CDC)**, **Outbox**

> Уровень: работа с БД, потоками данных, согласованностью.

### 4. **Паттерны безопасности (Security Patterns)**
Решают типовые задачи защиты приложений.
- Примеры:
    - **Authentication Gateway**, **Role-Based Access Control (RBAC)**, **Secure Logger**, **Tokenization**
> Уровень: аутентификация, авторизация, защита данных.

### 5. **Паттерны отказоустойчивости (Resilience / Fault Tolerance Patterns)**
Обеспечивают устойчивость системы к сбоям.
- Примеры:
    - **Circuit Breaker**, **Retry**, **Timeout**, **Bulkhead**, **Fallback**
> Уровень: взаимодействие между компонентами в условиях нестабильности.


### 6. **Паттерны облачной архитектуры (Cloud Design Patterns)**

Специфичны для облачных сред (AWS, Azure, GCP).
- Примеры:
    - **Strangler Fig**, **Ambient Intelligence**, **Auto-Scaling**, **Queue-Based Load Leveling**
> Уровень: использование возможностей облачных платформ.
