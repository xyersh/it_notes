
## Основные тезисы
- Вспомогательный код - это:
	- инициализации `config`, `logger`, `tracing`, `metrics`, `kafka`, `redis`, `DB` и т.д.
	- Обработка команд: `HTTP`, `GRPC`, `Consumer`, `Worker`, `CLI` и т.д.
	- Взаимодействие в внешним миром: `Postgres`, `Kafka`, `REdis` и т.д.
	- Библиотеки и прочее: `pkg`, `libs`, `middleware`, `sutdown`
	
- Основной код - это суть того что делает приложение (обычно вмещается в 1 экран).
	- Валидация входных данных
	- Работа с доменными моделями и функциями `domain`
	- Сеть: БД, Kafka, Redis, другой сервис
	- Транзакции в БД
	- Циклы, ветвления, параллельная обработка
	
- Разделяем основной код от вспомогательного

- Сервер рассматривается как  группа программ. Каждая ручка это отдельная программа. У каждой ручки СВОЙ код. Основной код и есть программа.
		- GET /api/v1/profile/{id}  Get profile
			- Валидация UUID - вызывается вспомогательный код для получения UUID
			- Получение профиля  из БД - вызывается вспом код для получения данных из БД
		- POST /api/v1/profile/{id} Create profile
			- Проверка дубликата в Redis - вызов вспом кода для проверки ключа идемпотентности
			- Валидация и создание доменной структуры - 
			- Сохранение профиля в БД
			- Отправляем событие в Kafka
 - У основного кода программы есть `контракт` это `DTO`: Input, Output. 
 - Не важно кто подает данные на вход основному коду - DTO буде единообразен
 - DTO - Data Tranfer Object

- Основной код должен быть:
	- Быть кратким - смещаться в 1 экран
	- Быть полным - не скрывать важные этапы
	- Показывать что происходит глобально
	- Не грузить деталями реализации
	- НЕ БЫТЬ частью другого основного кода

- 1 ручка = 1 программа = 1 основной код

- Кто вызывает основной код?
	- HTTP, GRPC
	- Kafka consumer
	- CRON worker
	- CLI команда

- Основной код не должен ничего знать о вызывающей стороне. 
- Артефакты вызывающей стороны не должны просачиваться в основной код

- Сторона, вызывающая основной код называется `CONTROLER`
- Сторона, вызываемая основным кодом называется `ADAPTER`
- Основной код называется `USECASE`
