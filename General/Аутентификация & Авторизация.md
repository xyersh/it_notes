## Аутентификация (Authentication)

**«Кто вы?»** — процесс проверки личности пользователя.

**Примеры:**
- Ввод логина и пароля
- Вход по отпечатку пальца или Face ID
- Использование одноразового кода из SMS или приложения (2FA)
- Вход через Google/Apple ID (OAuth)

## Авторизация (Authorization)

**«Что вам разрешено?»** — процесс проверки прав доступа к ресурсам.

**Примеры:**
- Обычный пользователь может просматривать статьи, но не может их удалять
- Администратор имеет доступ к панели управления
- Пользователь может редактировать только свои файлы

## важные термины в сфере доступа
| Термин                     | Расшифровка                          | Описание                                                             |
| -------------------------- | ------------------------------------ | -------------------------------------------------------------------- |
| **Login / Logon**          | Вход в систему                       | Процесс начала сессии после успешной аутентификации                  |
| **Logout / Logoff**        | Выход из системы                     | Завершение пользовательской сессии                                   |
| **Session**                | Сессия                               | Временный контекст взаимодействия пользователя с системой            |
| **Token**                  | Токен                                | Цифровой ключ (например, JWT), подтверждающий аутентификацию         |
| **SSO**                    | Single Sign-On                       | Единый вход: один логин для доступа к нескольким системам            |
| **MFA / 2FA**              | Multi/Two-Factor Authentication      | Многофакторная аутентификация: пароль + дополнительный код           |
| **OAuth / OpenID Connect** | Протоколы авторизации/аутентификации | Стандарты для безопасного входа через сторонние сервисы              |
| **RBAC**                   | Role-Based Access Control            | Управление доступом на основе ролей (пользователь, модератор, админ) |
| **ABAC**                   | Attribute-Based Access Control       | Доступ на основе атрибутов (время, местоположение, устройство)       |
| **Impersonation**          | Имитация                             | Возможность администратора действовать от имени другого пользователя |
| **Account Lockout**        | Блокировка учётной записи            | Временная блокировка после нескольких неудачных попыток входа        |
| **Password Policy**        | Политика паролей                     | Требования к сложности, сроку действия и истории паролей             |



## Как это работает вместе (упрощённый поток)
```
1. Пользователь вводит логин/пароль
   ↓
2. [Аутентификация] Система проверяет учётные данные
   ↓
3. Если успешно → создаётся сессия/токен
   ↓
4. [Авторизация] При каждом запросе система проверяет:
   - Валидна ли сессия?
   - Есть ли у пользователя право на это действие?
   ↓
5. Если всё ОК → доступ предоставлен, иначе → ошибка 403/401
```
