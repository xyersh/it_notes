
Префиксы в названии функций помогают понять намерение автора кода, не заглядывая внутрь функции. Вот самые популярные:

### Общие  правила использования префиксов
#### 1. Предикаты (Логические вопросы)

Возвращают логическое значение (`true`/`false`).
- **`is`**: Состояние объекта (`isAlive`, `isAdmin`).
- **`has`**: Наличие свойства или элемента (`hasLicense`, `hasAccess`).
- **`can`**: Возможность совершить действие (`canEdit`, `canPost`).
- **`should`**: Рекомендация или условие (`shouldRefresh`).

#### 2. Жизненный цикл и события
- **`on`**: Обработчик события (`onClick`, `onLoad`).
- **`handle`**: Логика обработки данных или ошибок (`handleRequest`, `handleError`).
- **`before` / `after`**: Хуки, выполняющиеся до или после основного действия.

#### 3. Получение и установка (Accessors)
- **`get`**: Возвращает значение. В некоторых языках подразумевает легкое вычисление или просто доступ к полю.
- **`set`**: Устанавливает значение.
- **`fetch` / `request`**: Подразумевает асинхронный запрос к API или базе данных (длительная операция).
- **`find`**: Поиск элемента (может вернуть `null`, если ничего не найдено).

#### 4. Трансформация и создание
- **`to`**: Конвертация из одного типа в другой (`toString`, `toJson`).
- **`as`**: Представление объекта в другом виде (`asArray`).
- **`new` / `create`**: Создание нового экземпляра объекта.
- **`parse`**: Разбор строки или данных в структуру.
    

### Правила Golang

| Префикс      | Назначение                                                                                                                                      |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `Must`       | Функция обязана выполниться успешно, иначе - паника                                                                                             |
| `New`        | Функция-конструктор:<br>**`New<Type>`**: Если пакет экспортирует несколько типов.<br>**`New`**: Если имя пакета уже описывает возвращаемый тип. |
| `Get`        | В голанг для геттеров префикс не испозуется                                                                                                     |
| `Set<Field>` | Сеттер                                                                                                                                          |
| `To<Type>`   | Преобразование данных в другой формат                                                                                                           |
| `As<Type>`   | Часто используется в интерфейсах или для приведения типов (type assertion).                                                                     |
| `Is...`      | Состояние объекта                                                                                                                               |
| `Has...`     | Для проверки наличия                                                                                                                            |
