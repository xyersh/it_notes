`PIVOT` поворачивает уникальные значения одного столбца в отдельные столбцы, одновременно применяя агрегатную функцию к другим числовым столбцам. Это классический инструмент для:

- Кросс-табличных отчетов
- Аналитических дашбордов
- Сведения нормализованных данных к плоскому виду
- Замены сложных конструкций `CASE + GROUP BY`


## СИНТАКСИС
```sql
SELECT <список_столбцов>
FROM <источник_данных>
PIVOT (
    <агрегатная_функция>(<столбец_значений>)
    FOR <столбец_для_поворота> IN (
        <значение1> AS <псевдоним1>,
        <значение2> AS <псевдоним2>,
        ...
    )
) [AS] <псевдоним_результата>
WHERE <условия>;
```

## Ключевые правила:
- **Обязательна агрегатная функция** (`SUM`, `COUNT`, `MAX`, `AVG`, `MIN` и т.д.)
- Все столбцы источника, **не указанные** в `FOR` и **не являющиеся** аргументом агрегации, становятся **группирующими** (implicit `GROUP BY`)
- Список в `IN` **статический**. Для динамических столбцов используется `PIVOT XML` или динамический SQL
- Отсутствующие комбинации возвращают `NULL`


## ПРИМЕР

Подготовка данных
```sql
-- Создание таблицы
CREATE TABLE sales_data (
    manager_id NUMBER,
    quarter    VARCHAR2(10),
    amount     NUMBER
);

-- Заполнение данными
INSERT ALL
    INTO sales_data VALUES (101, 'Q1', 1200)
    INTO sales_data VALUES (101, 'Q2', 1800)
    INTO sales_data VALUES (101, 'Q3', 1500)
    INTO sales_data VALUES (102, 'Q1',  900)
    INTO sales_data VALUES (102, 'Q2', 1100)
    INTO sales_data VALUES (102, 'Q4', 1300)
    INTO sales_data VALUES (103, 'Q1', 1400)
    INTO sales_data VALUES (103, 'Q3', 1600)
    INTO sales_data VALUES (103, 'Q4', 1700)
SELECT * FROM DUAL;

COMMIT;
```

Запрос PIVOT
```sql
SELECT *
FROM sales_data
PIVOT (
    SUM(amount)
    FOR quarter IN ('Q1' AS q1, 'Q2' AS q2, 'Q3' AS q3, 'Q4' AS q4)
)
ORDER BY manager_id;
```

 Подробное объяснение кода
 
 | Часть запроса | Что делает |
|---|---|
|`SUM(amount)`| Агрегирует значения столбца `amount`. Если для одного менеджера и квартала несколько строк, они суммируются. |



|`FOR quarter IN (...)`| Указывает, какие значения столбца `quarter` станут новыми столбцами. Без `AS` имена были бы `'Q1'`, `'Q2'` (с кавычками), что неудобно для дальнейшего использования.|
|Неявная группировка | Поскольку в источнике три столбца, а `amount` аггрегируется, а `quarter` поворачивается, Oracle автоматически группирует по оставшемуся `manager_id`.|
|`ORDER BY manager_id`|Сортировка применяется **после** `PIVOT`, так как это внешняя операция к блоку `PIVOT`.|
